# Available Features & Capabilities

## What You Can Do in iPerform

This guide explains all the features available in iPerform - organized by what you want to accomplish, not by technical details.

## Authentication & Account Management

### Logging In

**What You Need**:
- Your email address
- Your password

**What Happens**:
- System verifies your credentials
- Creates a secure session
- Loads your personalized dashboard
- Shows only information relevant to your role

**Session Features**:
- Stays logged in for 90 days (unless you log out)
- Can access from multiple devices
- View all your active sessions
- Remotely log out from other devices

### Managing Your Profile

**What You Can Update**:
- Personal information (name, phone, email)
- Password
- Notification preferences
- Display settings
- Language preferences

## Lead Management Features

### Viewing Leads

**What You Can See**:
- Leads assigned to you (or all leads if you're a manager/admin)
- Company name and contact information
- Current status in the sales pipeline
- Assigned salesperson/warrior
- Source of the lead
- Complete activity history
- Upcoming follow-up tasks

**Filtering Options**:
- By status (Open, Connected, Won, etc.)
- By source (Facebook, Website, Partner, etc.)
- By date range
- By assigned person
- By industry
- By location

**Sorting Options**:
- Newest first
- Oldest first
- By company name
- By last activity
- By priority

### Creating New Leads

**Methods Available**:

**1. Manual Entry**:
Simply fill out a form with customer details:
- Company name
- Contact person name
- Email address
- Phone number
- Location
- Industry
- How they found you (source)
- Initial notes

**2. Bulk Upload from Excel**:
- Download Excel template
- Fill in lead information (hundreds at once)
- Upload the file
- System creates all leads automatically
- Shows success/failure report

**3. Facebook Automatic Capture**:
- Connect your Facebook account (one-time setup)
- Leads automatically created when someone fills your Facebook ad form
- No manual work needed!

**4. Partner Submission**:
- Partners submit leads through their portal
- Leads automatically appear in your system
- Proper attribution to partner maintained

### Updating Leads

**What You Can Change**:
- Lead status (move through pipeline)
- Contact information
- Notes and comments
- Assigned salesperson/warrior
- Priority level
- Industry or category
- Custom fields

**Status Update**:
When you update status, you can also:
- Add reason for status change
- Add detailed notes
- Attach documents
- Schedule next follow-up
- Set reminders

### Lead Assignment

**Auto-Assignment** (Administrators/Managers):
- System calculates workload for each team member
- Automatically assigns to person with fewest leads
- Distributes leads fairly
- Can be customized by territory or expertise

**Manual Assignment**:
- Choose specific salesperson/warrior
- Transfer leads between team members
- Reassign if someone leaves
- Bulk reassignment available

### Follow-Up Management

**Schedule Follow-Ups**:
- Set reminder date and time
- Choose communication type (Call, Email, WhatsApp, Meeting)
- Add notes about what to discuss
- System will remind you at the right time

**View Your Follow-Ups**:
- See today's follow-ups on dashboard
- View upcoming follow-ups
- Mark as complete when done
- Snooze/reschedule if needed

## Support Ticketing

### Creating Tickets

**When to Create a Ticket**:
- Customer reports a problem
- Technical issue discovered
- Feature request from customer
- Integration needed
- Customization required
- Any customer service issue

**Ticket Information**:
- Link to customer/lead
- Issue type (Sync, Bug, Feature Request, etc.)
- Priority (Normal, High, Critical)
- Detailed description
- Platform affected (Website, Mobile, Both)
- Communication channel (How customer contacted you)

### Managing Tickets

**Assign Tickets**:
- To specific team member
- To technical team
- To customer success team
- Automatically routed based on issue type

**Add Comments**:
- Update progress
- Ask questions
- Share findings
- Document resolution steps
- Visible to whole team

**Update Status**:
Move ticket through stages:
- Assigned → Someone is responsible
- In Process → Actively working on it
- Pending Customer → Waiting for customer info
- Resolved → Solution found
- Deployed → Fix is live
- Closed → Issue completely done

### Viewing Tickets

**Your Tickets**:
- Tickets assigned to you
- Tickets you created
- Tickets for your customers

**Filter Tickets**:
- By status
- By priority
- By issue type
- By assigned person
- By date range

**Ticket Dashboard**:
- Open tickets count
- Resolved today
- Average resolution time
- Your pending tickets

## Access Levels Summary

### What Each Role Can Do

**Administrators**:
✓ Everything  
✓ Manage all users  
✓ Change all settings  
✓ View all data  
✓ Generate all reports  

**Managers**:
✓ View team data  
✓ Assign leads  
✓ Team reports  
✓ Performance monitoring  
✗ Cannot change system settings  

**Salespersons**:
✓ Manage own leads  
✓ Send communications  
✓ Create tickets  
✓ View own reports  
✗ Cannot reassign leads  
✗ Cannot see other's leads  

**Warriors**:
✓ Update assigned leads  
✓ Mark won/lost  
✓ Add notes  
✗ Cannot see tickets  
✗ Limited status changes  

**Technical Team**:
✓ Manage tickets  
✓ Technical settings  
✓ View customer tech data  
✗ Limited lead access  

**Guests**:
✓ View reports only  
✗ Cannot edit anything  
✗ No customer details
